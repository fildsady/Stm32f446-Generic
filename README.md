# STM32F446 Generic Template

แม่แบบโปรเจกต์ (Project Template) สำหรับบอร์ดไมโครคอนโทรลเลอร์ STM32F446RE ที่เน้นความสะอาดของโค้ดและการดึงประสิทธิภาพของฮาร์ดแวร์ออกมาให้ได้มากที่สุด โดยไม่พึ่งพาระบบ Auto-generate ของ STM32CubeMX

โปรเจกต์นี้ถูกออกแบบมาเพื่อเป็นรากฐานที่แข็งแกร่งสำหรับการพัฒนาเฟิร์มแวร์ระดับมืออาชีพ โดยเฉพาะอย่างยิ่งงานที่เกี่ยวข้องกับเสียง (Audio) หรือการประมวลผลสัญญาณดิจิทัล (Digital Signal Processing)

## คุณสมบัติเด่น (Features)

- **STM32 LL API (Low-Layer API):** แทนที่ไลบรารี HAL แบบดั้งเดิมด้วยไลบรารี LL API อย่างเป็นทางการจาก STMicroelectronics เพื่อการควบคุมรีจิสเตอร์ (Register) ที่รวดเร็วและใช้หน่วยความจำน้อยที่สุด
- **ระบบบิลด์ CMake และ Ninja:** โครงสร้างการคอมไพล์ที่รวดเร็วแบบ Out-of-source build ไม่สร้างไฟล์ขยะปะปนกับซอร์สโค้ด และทำงานร่วมกับ Toolchain จาก STM32CubeCLT ได้อย่างสมบูรณ์บนระบบปฏิบัติการ Windows
- **ระบบปฏิบัติการเวลาจริง (RTOS):** ติดตั้งไลบรารี FreeRTOS-Kernel แบบ Submodule พร้อมใช้งานสำหรับการทำ Multitasking ทันที
- **ไลบรารีประมวลผลสัญญาณ (CMSIS-DSP):** (ใน Branch `feature/cmsis-dsp`) รองรับคำสั่งทางคณิตศาสตร์ความเร็วสูงที่ดึงพลังของหน่วยประมวลผลจุดทศนิยม (FPU) แบบฮาร์ดแวร์ และชุดคำสั่ง SIMD บนสถาปัตยกรรม Cortex-M4F มาใช้ เช่น `arm_mult_f32`, `arm_sin_f32`, FIR, IIR และ FFT

## โครงสร้างโปรเจกต์ (Project Structure)

- `Drivers/` - เก็บไลบรารีพื้นฐานและ Submodule ที่จำเป็น
  - `CMSIS/` - ไลบรารีแกนหลักของสถาปัตยกรรม ARM Cortex
  - `STM32F4xx_HAL_Driver/` - ไลบรารีของ ST (ใช้งานเฉพาะส่วนที่เป็น LL API)
  - `CMSIS-DSP/` - ไลบรารีคณิตศาสตร์ (เฉพาะ Branch `feature/cmsis-dsp`)
- `FreeRTOS-Kernel/` - ซอร์สโค้ดของระบบปฏิบัติการ FreeRTOS
- `Inc/` - โฟลเดอร์เก็บไฟล์ Header (`.h`) ของโปรเจกต์
- `Src/` - โฟลเดอร์เก็บไฟล์ Source (`.c`, `.S`)
- `cmake/` - สคริปต์สำหรับการตั้งค่า Toolchain ของ CMake

## วิธีการบิลด์ (How to Build)

โปรเจกต์นี้ตั้งค่ามาเพื่อให้ทำงานได้อย่างราบรื่นกับ **VS Code** และ **STM32 VS Code Extension**

1. ตรวจสอบว่าในระบบมีการติดตั้ง **STM32CubeCLT** เรียบร้อยแล้ว
2. เปิดโฟลเดอร์โปรเจกต์นี้ด้วย VS Code
3. เปิด Command Palette (`Ctrl + Shift + P`):
   - พิมพ์และเลือกคำสั่ง `CMake: Configure` เพื่อตั้งค่าโปรเจกต์ในครั้งแรก
   - พิมพ์และเลือกคำสั่ง `CMake: Build` (หรือกดคีย์ลัด `Ctrl + Shift + B` หากตั้งค่า Tasks ไว้) เพื่อทำการคอมไพล์
4. ไฟล์ไบนารีสำหรับนำไปแฟลชลงบอร์ด (`.elf`, `.bin`, `.hex`) จะถูกสร้างและจัดเก็บไว้ในโฟลเดอร์ `build/Debug/`

## ข้อมูลสภาพแวดล้อม (Environment Details)

- **ชิปเป้าหมาย:** STM32F446RET6
- **ความถี่สัญญาณนาฬิการะบบ (System Clock):** 180 MHz (ตั้งค่าในโค้ดผ่าน LL API)
- **คอมไพเลอร์:** `arm-none-eabi-gcc`

## การทำงานของ CMSIS-DSP (เฉพาะ Branch `feature/cmsis-dsp`)

ในไฟล์ `main.c` ของ Branch นี้ มีการเพิ่มฟังก์ชัน `dsp_test_task` ซึ่งเป็น Task ของ FreeRTOS สำหรับทดสอบการทำงานของไลบรารี CMSIS-DSP โดยเฉพาะ โค้ดส่วนนี้แสดงให้เห็นถึงการดึงพลังของหน่วยประมวลผลจุดทศนิยม (FPU) ออกมาใช้งานจริง:

1. **การคูณอาร์เรย์ (Vector Multiplication):**
   ```c
   arm_mult_f32(a, b, result, 4);
   ```
   ฟังก์ชันนี้ใช้สำหรับคูณข้อมูลในอาร์เรย์ `a` และ `b` เข้าด้วยกันแล้วเก็บผลลัพธ์ลงใน `result` การทำงานภายใต้ไลบรารีนี้ไม่ได้ใช้ลูป `for` ธรรมดา แต่ใช้ชุดคำสั่ง **SIMD (Single Instruction, Multiple Data)** ของสถาปัตยกรรม Cortex-M4F ซึ่งสามารถประมวลผลข้อมูลหลายตัวพร้อมกันใน 1 สัญญาณนาฬิกา (Clock cycle) เหมาะสำหรับการปรับระดับเสียง (Volume control) หรือการมิกซ์เสียง

2. **การคำนวณตรีโกณมิติแบบรวดเร็ว (Fast Math):**
   ```c
   float32_t sin_val = arm_sin_f32(PI/4);
   ```
   ฟังก์ชันนี้ใช้ตารางค้นหา (Lookup Table) ร่วมกับการประมาณค่าในช่วง (Interpolation) ที่ปรับแต่งมาเพื่อฮาร์ดแวร์ ARM โดยเฉพาะ ทำให้สามารถคำนวณค่า Sine ได้เร็วกว่าการใช้ฟังก์ชัน `sin()` จากไลบรารีมาตรฐาน `<math.h>` ของภาษา C หลายเท่า เหมาะสำหรับการสร้างเสียงสังเคราะห์ (Synthesizer) หรือการสร้างคลื่นความถี่อ้างอิง
